⬆️ 取枪流程（PICK）。放枪流程（PLACE）是其镜像：ARM_TO_PLACE → MOTOR_RELEASE → SERVO_RETRACT → CLAW_OPEN → ARM_RETRACT → DONE

3.1 取枪子状态机 (Pick Flow)

状态名称动作完成条件超时(ms)
P0 IDLE 无动作，等待触发命令 收到 pick 命令 —
P1 ARM_TO_PICK 机械臂运动到取枪位置 (mode=2, s=0.39~0.65, h=0.14)
同时抬升 R2 lift 到避让位 (11.5)		 三关节到位 & R2 lift 到位 5000
P2 PRE_CLAW_OPEN KFS夹爪张开 (kfs_catch → 35.0) kfs_catch.position > 34.0 2000
P3 CLAW_CLOSE KFS夹爪闭合抓取枪杆 (kfs_catch → 0) kfs_catch.position < 1.0 3000
P4 SLIDE_PUSH TIM2 CH3 → 600us (滑台推出) 延时 300ms 1000
P5 SERVO_UPRIGHT TIM2 CH1 → 2100us (舵机推杆竖立) 延时 400ms 1000
P6 MOTOR_COLLECT 2325电机旋转收集 (hold_weapon: pos=-1.5, kp=1.2, kd=0.2) weapon_collect_motor.position < -1.0 3000
P7 SERVO_MID TIM2 CH1 → 1600us (舵机回中位) 延时 200ms 500
P8 SLIDE_RETRACT TIM2 CH3 → 2500us (滑台收回) 延时 300ms 1000
P9 ARM_RETRACT 机械臂回安全位 (arm_1=0.4, arm_2=-0.39, arm_3=23.8) 三关节到位 5000
P10 DONE 清除标志，返回 IDLE — —

3.2 放枪子状态机 (Place Flow) — 取枪的镜像

状态名称动作完成条件超时(ms)
L0 IDLE 等待 place 命令 收到命令 —
L1 ARM_TO_PLACE 机械臂到放枪位 (takePos=1: arm_1=0.4, arm_2=-0.55, arm_3=23.8)
R2 lift 到 0.2 (最低)		 到位 5000
L2 SLIDE_PUSH TIM2 CH3 → 600us 延时 300ms 1000
L3 SERVO_RELEASE TIM2 CH1 → 1450us (舵机释放) 延时 400ms 1000
L4 MOTOR_RELEASE 2325电机反向释放 (hold_weapon 回零) weapon_collect_motor.position > -0.2 3000
L5 CLAW_OPEN KFS夹爪张开释放枪杆 kfs_catch.position > 34.0 2000
L6 ARM_RETRACT 机械臂收回 到位 5000
L7 DONE 完成 — —

3.3 换枪子状态机 (Change Flow)

换枪 = 放枪(前半) + 取枪(后半)，无机械臂移回安全位
状态名称动作条件
C0 RELEASE_OLD SLIDE_PUSH → SERVO_RELEASE → MOTOR_RELEASE → CLAW_OPEN 各阶段到位
C1 PICK_NEW CLAW_CLOSE → SLIDE_PUSH → SERVO_UPRIGHT → MOTOR_COLLECT → SERVO_MID → SLIDE_RETRACT 各阶段到位

🔄 4. 核心子流程详解

SUB-FLOW 机械臂取枪运动
// 机械臂运动到取枪位置 (mode=2: 取weapon)
Arm_Info_TypeDef arm;
arm.mode = 2;
arm.s    = 0.39f;  // 水平距离 (可通过accel调节0.39~0.65)
arm.h    = 0.14f;  // 垂直高度
angles = Arm_Inverse_Solution(&arm);
Torque = Torque_Comp_global(&kfs_arm_1, &kfs_arm_2, &kfs_arm_3);

// 设置三关节目标
kfs_arm_1.set_mit_data(&kfs_arm_1,
    angles.theta_1 + 0.45f, 0.0f,
    20.0f, 1.5f, -0.25f * Torque.Torque_1);
kfs_arm_2.set_mit_data(&kfs_arm_2,
    -angles.theta_2, 0.0f,
    30.0f, 2.0f, -Torque.Torque_2);
kfs_arm_3.set_mit_data(&kfs_arm_3,
    -25 * (-kfs_arm_2.position - kfs_arm_1.position
    + 0.43 + a_weapon + 0.2f * PI),
    0.0f, 3.0f, 1.5f, Torque.Torque_3);

// 到位判断 (参考 auto_mission):
// arm_1: 0.48~0.52, arm_2: -0.33~-0.29, arm_3: -23~-19
SUB-FLOW 2325电机收集/释放
// === 收集 (Collect) ===
float hold_weapon[5];
hold_weapon[0] = -1.5f;  // 位置目标
hold_weapon[1] = 0.0f;   // 速度
hold_weapon[2] = 1.2f;   // kp
hold_weapon[3] = 0.2f;   // kd
hold_weapon[4] = 0.0f;   // torque feedforward
weapon_collect_motor.set_mit_data(
    &weapon_collect_motor,
    hold_weapon[0], hold_weapon[1],
    hold_weapon[2], hold_weapon[3], hold_weapon[4]);

// === 释放 (Release) ===
hold_weapon[0] = 0.0f;
hold_weapon[1] = 0.8f;  // 低速回零
hold_weapon[2] = 0.0f;
hold_weapon[3] = 0.3f;
hold_weapon[4] = 0.0f;

// === 收紧 (Tighten, 最终位) ===
hold_weapon[0] = 0.0f;
hold_weapon[4] = 0.9f;  // 持续力矩保持

7.2 调度方式

方案A: FreeRTOS 独立任务

// 创建独立任务，优先级低于运动控制
osThreadNew(ClampRodAuto_Task, NULL, &clamp_rod_auto_attributes);
// 优点: 独立栈空间，可阻塞等待
// 缺点: 多一个任务，增加调度开销

✅ 方案B: 在 Free_Task 中轮询 (推荐)

// 在 Free_Task() 循环中添加:
if (g_clamp_rod.state != CLAMP_ROD_STATE_IDLE) {
    ClampRodAuto_Run();
} else {
    // 检测触发条件
    if (BUT_flag.button_new == 1) {
        ClampRodAuto_StartChange();
    }
}
// 优点: 零额外开销，复用现有任务
// 缺点: 每个周期只执行一步 (这正好是状态机特性)

7.3 安全保护

保护机制实现
阶段超时每个状态有独立超时 (1~5s)，超时→ERROR→IDLE恢复
总流程超时pick 15s / place 13s，防止无限等待
遥控器失联RCctrl.rc_lost → 立即 Abort() → 机械臂保持+夹爪松开
传感器检查PE9 在关键阶段验证枪杆状态，不匹配则重试或报错
软限位保护各电机位置在 arm/weapon/kfs 层已有范围限制
重试机制retry_count ≤ 3，超阈值 → ERROR 需人工介入
